# QQ-MC验证插件

QQ群验证我的世界账号插件，实现跨平台账号绑定。

## 功能特性

- 玩家在MC服务器输入 `.rq` 获取验证码
- 玩家在QQ群发送 `/验证 验证码` 完成绑定
- 一个QQ号只能绑定一个MC账号
- 验证失败3次将被限制
- 验证成功后MC端自动发放积分奖励
- 支持黑名单机制

## 安装

1. 将 `qq_mc_verify` 文件夹放入 AstrBot 的 `addons/plugins/` 目录
2. 重启 AstrBot 或在管理后台重新加载插件
3. 在插件配置中设置相关参数

## 配置说明

| 配置项 | 说明 | 默认值 |
|--------|------|--------|
| web_port | Web服务端口（供MC端连接） | 9090 |
| mc_bot_url | MC端Web地址（通过frp映射的地址） | http://localhost:9091 |
| log_retention_days | 日志保留天数 | 30 |
| clear_database | 一键清除数据库（慎用） | false |

## 指令说明

### 玩家指令

| 指令 | 说明 |
|------|------|
| `/验证 验证码` | 验证MC账号 |
| `/验证状态` | 查看当前验证状态 |

### 管理员指令

| 指令 | 说明 |
|------|------|
| `/验证列表` | 查看待验证列表 |
| `/验证记录` | 查看最近验证记录 |
| `/清除数据库` | 一键清除所有数据 |

## 使用流程

1. 玩家在MC服务器中输入 `.rq`
2. 系统生成验证码并显示给玩家
3. 玩家在QQ群发送 `/验证 验证码`
4. 验证成功后，QQ号与MC账号绑定
5. MC服务器自动发放积分奖励

## API接口

### POST /api/mc_verify_request

MC端发送验证请求

请求体：
```json
{
    "code": "验证码",
    "player_name": "玩家名",
    "xuid": "玩家XUID",
    "unique_id": "玩家唯一ID"
}
```

### POST /api/check_binding

检查玩家是否已绑定

请求体：
```json
{
    "player_name": "玩家名"
}
```

### GET /api/status

检查服务状态

### GET /api/pending

获取待验证列表

### GET /api/records

获取验证记录

### GET /api/logs

获取日志记录

## 注意事项

1. 确保端口已正确映射（Docker需要端口映射）
2. MC端需要配置正确的QQ端地址
3. QQ端需要配置正确的MC端地址（通过frp映射）
4. 验证码有效期为3分钟
